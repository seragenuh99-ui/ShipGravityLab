# CpuAggregator.cpp - CPU汇总计算模块

## 文件概述
这是ShipGravityLab项目的CPU汇总模块，负责将所有GPU计算得到的单元属性汇总为整体重量和重心。

## 核心功能
1. 累加所有单元的重量
2. 计算总重量矩（重量 × 形心坐标）
3. 计算整体重心坐标（总重量矩 / 总重量）
4. 提供简单高效的CPU参考实现

## 数学原理

### 重心计算公式
```
总重量 W = Σ(w_i)                // 所有单元重量之和
总重量矩 M = Σ(w_i * c_i)        // 重量乘以形心坐标
整体重心 C = M / W               // 总重量矩除以总重量

其中：
  w_i = 第i个单元的重量
  c_i = 第i个单元的形心坐标(x,y,z)
  W = 总重量
  M = 总重量矩（三维向量）
  C = 整体重心坐标(x,y,z)
```

### 物理意义
- **重量累加**：船舶总重量等于所有结构部件重量之和
- **重量矩**：每个部件对整体重心的贡献与其重量和位置有关
- **加权平均**：整体重心是各部件重心的加权平均，权重为部件重量

## 代码结构详解

### 1. 主函数接口
```cpp
void calcTotalGravityCenter(
    const ShipPlateGpuData* plates,   // 板数据数组
    int plateCount,                   // 板数量
    const ShipBeamGpuData* beams,     // 梁数据数组  
    int beamCount,                    // 梁数量
    float& totalWeight,               // 输出：总重量
    float totalCentroid[3]            // 输出：重心坐标(x,y,z)
)
```
- **输入参数**：
  - `plates`：GPU计算后的板数据
  - `plateCount`：板单元数量
  - `beams`：GPU计算后的梁数据
  - `beamCount`：梁单元数量
- **输出参数**：
  - `totalWeight`：总重量（千克）
  - `totalCentroid`：三维重心坐标（米）
- **AI操作点**：可以扩展输出更多统计信息

### 2. 累加算法实现
```cpp
// 初始化累加器
totalWeight = 0;
totalCentroid[0] = 0;  // X方向总重量矩
totalCentroid[1] = 0;  // Y方向总重量矩  
totalCentroid[2] = 0;  // Z方向总重量矩

// 累加板的贡献
for (int i = 0; i < plateCount; i++) {
    totalWeight += plates[i].weight;
    totalCentroid[0] += plates[i].weight * plates[i].centroid[0];
    totalCentroid[1] += plates[i].weight * plates[i].centroid[1];
    totalCentroid[2] += plates[i].weight * plates[i].centroid[2];
}

// 累加梁的贡献（类似代码）
```
- **顺序累加**：简单直接的累加算法
- **数值稳定性**：可能存在累积误差
- **AI操作点**：可以改用Kahan求和算法提高精度

### 3. 重心计算
```cpp
// 计算整体重心坐标
if (totalWeight != 0.0f) {
    totalCentroid[0] /= totalWeight;  // X坐标
    totalCentroid[1] /= totalWeight;  // Y坐标
    totalCentroid[2] /= totalWeight;  // Z坐标
}
// 如果总重量为0，重心坐标保持为[0,0,0]
```
- **除零保护**：检查总重量是否为0
- **单位**：坐标单位为米，重量单位为千克
- **AI操作点**：可以添加更多错误检查

## 性能特征

### 当前性能
- 计算时间：约0.65ms（160万单元）
- 内存访问：顺序访问，缓存友好
- CPU占用：单线程
- 不是性能瓶颈

### 复杂度分析
- **时间复杂度**：O(N)，N为单元总数
- **空间复杂度**：O(1)，只使用几个累加变量
- **内存访问**：顺序读取，预取友好

## 数值考虑

### 浮点精度
- **单精度浮点**：使用float类型
- **累积误差**：大量累加可能产生误差
- **改进方案**：使用双精度或补偿求和

### 除零处理
- **当前处理**：简单检查totalWeight != 0
- **改进方案**：添加更严格的数值检查
- **边界情况**：极小的总重量处理
