# OperatorTags.h - 数据结构定义文件

## 文件概述
这是ShipGravityLab项目的核心数据结构定义文件，定义了GPU计算所需的所有数据结构和类型标签。

## 核心功能
1. 定义GPU计算数据结构
2. 定义单元类型标签和枚举
3. 提供跨模块共享的数据类型
4. 确保数据布局兼容CUDA内存对齐要求

## 数据结构详解

### 1. GPU计算数据结构

#### ShipPlateGpuData（板单元GPU数据）
```cpp
struct ShipPlateGpuData {
    // 顶点坐标（最多4个顶点）
    float p1[3];  // 顶点1坐标 (x,y,z)
    float p2[3];  // 顶点2坐标
    float p3[3];  // 顶点3坐标  
    float p4[3];  // 顶点4坐标（四边形时使用）
    
    // 单元属性
    int nodeCount;     // 节点数量：3=三角形，4=四边形
    float thickness;   // 板厚度（米）
    float density;     // 材料密度（kg/m³）
    
    // 计算结果（由GPU填充）
    float area;        // 面积（平方米）
    float centroid[3]; // 形心坐标 (x,y,z)
    float volume;      // 体积（立方米）
    float weight;      // 重量（千克）
};
```
- **内存布局**：152字节（假设float=4字节，int=4字节）
- **对齐要求**：CUDA要求适当对齐以提高内存访问效率
- **AI操作点**：可以调整数据成员顺序优化缓存

#### ShipBeamGpuData（梁单元GPU数据）
```cpp
struct ShipBeamGpuData {
    // 端点坐标
    float p1[3];  // 端点1坐标
    float p2[3];  // 端点2坐标
    
    // 梁属性
    float sectionArea;  // 截面面积（平方米）
    float density;      // 材料密度（kg/m³）
    
    // 计算结果
    float length;       // 长度（米）
    float centroid[3];  // 形心坐标（中点）
    float volume;       // 体积（立方米）
    float weight;       // 重量（千克）
};
```
- **内存布局**：64字节
- **简化设计**：梁计算比板计算简单，数据结构也相应简化
- **AI操作点**：可以添加更多梁属性如截面形状参数

## 设计原则

### 1. 数据驱动设计
- **统一数据结构**：所有板单元使用相同结构，通过nodeCount区分
- **简化接口**：减少函数参数数量
- **易于序列化**：连续内存布局便于传输

### 2. GPU友好设计
- **数组形式**：坐标使用float[3]数组而非独立变量
- **连续存储**：适合CUDA合并内存访问
- **适当大小**：避免过大的结构体影响寄存器使用

### 3. 扩展性设计
- **预留字段**：p4字段为四边形板预留
- **通用属性**：厚度、密度等通用属性
- **计算结果**：统一的计算结果字段

## 数据流分析

## 设计决策记录

### 决策1：使用float而非double
- **理由**：GPU单精度性能更好，内存占用减半
- **影响**：可能损失一些精度
- **验证**：船舶工程精度要求通常满足

### 决策2：统一板数据结构
- **理由**：简化代码，减少条件判断
- **影响**：四边形板浪费一个顶点存储
- **权衡**：代码简洁性 vs 内存效率

### 决策3：分离板和梁结构
- **理由**：计算差异大，统一结构更复杂
- **影响**：需要两种核函数
- **权衡**：计算效率 vs 代码重复
